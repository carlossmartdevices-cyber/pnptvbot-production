API ENDPOINTS SUMMARY
================================================================================

PUBLIC ENDPOINTS (3):
1. POST /api/coming-soon/waitlist
   - Sign up for feature waitlist
   - Returns: signup confirmation or existing entry

2. GET /api/coming-soon/count/:feature
   - Get total pending waitlist count
   - Returns: current count

3. POST /api/coming-soon/unsubscribe
   - Unsubscribe from waitlist
   - Returns: success confirmation

ADMIN ENDPOINTS (4) - Require Authentication:
4. GET /api/coming-soon/stats/:feature
   - View waitlist statistics
   - Returns: total, pending, notified, unsubscribed counts

5. GET /api/coming-soon/pending/:feature
   - View pending entries with pagination
   - Returns: list of email entries

6. POST /api/coming-soon/notify
   - Mark entries as notified
   - Returns: count of updated entries

7. GET /api/coming-soon/export/:feature
   - Export waitlist as CSV
   - Returns: CSV file download

================================================================================
DATABASE SCHEMA
================================================================================

TABLE: coming_soon_waitlist
- id (PRIMARY KEY)
- email (VARCHAR 255, NOT NULL)
- feature_type (VARCHAR 50, NOT NULL) - 'live' or 'hangouts'
- signed_up_at (TIMESTAMP)
- notified_at (TIMESTAMP)
- status (VARCHAR 20) - 'pending', 'notified', 'unsubscribed'
- source (VARCHAR 100) - signup source tracking
- user_id (FK to users, nullable)
- ip_address (INET)
- user_agent (TEXT)
- created_at (TIMESTAMP)
- updated_at (TIMESTAMP)
- UNIQUE(email, feature_type)

TABLE: coming_soon_waitlist_audit
- id (PRIMARY KEY)
- waitlist_id (FK to coming_soon_waitlist)
- action (VARCHAR 50) - 'signed_up', 'notified', 'unsubscribed'
- details (JSONB)
- created_at (TIMESTAMP)

VIEW: coming_soon_waitlist_stats
- Aggregated stats by feature_type
- Shows: total, pending, notified, unsubscribed counts
