Security Software Warnings

Why does my security software warn me?

Security software is designed to be cautious about camera access and sending images to external services. This is actually a good sign - it means your security is working! Age verification systems naturally trigger these warnings because they:

  • Request camera access (legitimate privacy concern)
  • Send images to AI services (legitimate security concern)
  • Process biometric data (legitimate regulation under GDPR)
These warnings are normal and expected for any legitimate age verification system.

Is this page malware?

No. The warning is a false positive. Here's how you can tell:

  • Valid HTTPS certificate for pnptv.app
  • No code obfuscation or hiding
  • Clear, transparent data handling
  • No suspicious network behavior
  • No file system modifications
  • Legitimate business purpose (age verification)

Should I allow camera access?

Only if you feel comfortable. The system is designed with multiple privacy protections:

  • You see a consent modal explaining everything first
  • You can always click "Decline" to refuse
  • Camera only activates AFTER you click "Accept"
  • You can use manual verification instead (no camera needed)
  • You control when the camera captures photos

Can I whitelist this site?

Yes. If you trust pnptv.app, you can:

  • Windows: Allow in Windows Defender SmartScreen
  • Mac: Allow in macOS Gatekeeper settings
  • Browser: Add pnptv.app to trusted sites
  • Antivirus: Add pnptv.app to whitelist in security software

Privacy & Data Handling

What happens to my photo?

It's deleted immediately. Here's the flow:

  1. You take a selfie in your browser
  2. Photo is sent to Face++ AI (encrypted via HTTPS)
  3. Face++ analyzes it in real-time
  4. Face++ deletes the photo from its servers
  5. Our server deletes the photo from memory
  6. Garbage collection clears the memory buffer
Result: Photo is gone within seconds. We never store it.

What data IS stored?

Only verification metadata:

  • Estimated age (from AI analysis)
  • Confidence score (how certain the AI is)
  • Verification date
  • Which AI provider was used
What's NOT stored:
  • No facial images
  • No photos
  • No facial recognition data
  • No personal identification data

How long is my data kept?

Verification expires after 7 days. After that:

  • You'll need to re-verify if accessing age-restricted features
  • Old verification records are automatically deleted
  • You can request manual deletion at any time

Is my data sold to third parties?

No. Your data is:

  • Never sold
  • Never shared with advertisers
  • Never used for marketing
  • Only used for age verification
  • Protected under our Privacy Policy

Technical Details

What's Face++?

Face++ (also called Megvii) is a legitimate AI company specializing in facial analysis. They:

  • Serve 50+ million users worldwide
  • Focus on age estimation (not facial recognition)
  • Are ISO/IEC 27001 certified
  • Follow GDPR and data protection standards
  • Do NOT store facial images sent via API

Is the connection secure?

Yes. All connections are:

  • HTTPS encrypted (TLS 1.2+)
  • Certificate verified
  • Protected against man-in-the-middle attacks
  • Using modern encryption standards

What security headers are in place?

The system uses multiple security headers:

  • X-Content-Type-Options: nosniff - Prevents MIME type attacks
  • X-Frame-Options: SAMEORIGIN - Prevents clickjacking
  • X-XSS-Protection - Blocks XSS attacks
  • Content-Security-Policy - Strict limits on scripts
  • Permissions-Policy - Limits camera/microphone access

Not Comfortable With Camera?

Can I skip AI verification?

Yes! You have two options:

  1. Click "Decline" on the consent modal - Returns to main menu
  2. Click "Skip to Manual Verification" - Confirm age manually
Manual verification:
  • No camera access required
  • No photos taken
  • No external APIs
  • Just confirm you're 18+

What if the camera doesn't work?

If your camera isn't working:

  1. Check browser camera permissions (usually in address bar)
  2. Refresh the page and try again
  3. Use a different browser that has camera support
  4. Use manual verification as fallback
